# Sprint 7 — v1.1 Foundation Plan

> **Goal:** Ship v1.1 of openDesk Edu — DFN-AAI federation support, automated user provisioning via the existing `user_import` tooling, and semester lifecycle management deployed on-cluster.

**Current State:**
- v1.0.1 hardening (Sprint 5-6) is shipped: SSO audit, domain migration, k8up, Grafana, Portal SAML multidomain fix
- `scripts/user_import/` has working Python provisioning/deprovisioning code (UCS UDM API + Keycloak) with a Dockerfile
- `helmfile/apps/semester-provisioning/` has a complete Helm chart with templates, but references a non-existent image (`semester-provisioning:latest`)
- DFN-AAI setup scripts exist in `scripts/dfn-aai-setup/` — not yet applied to cluster
- SLO doc exists (`docs/backchannel-logout.md`) — backchannel logout not verified
- Semester lifecycle management docs exist (`docs/semester-lifecycle.md`, `docs/semester-automation-guide.md`)

**Tech Stack:** Python 3.12, Helm, K3s, UCS UDM REST API, Keycloak Admin API, DFN-AAI SAML, Labeller/GitLab CI

### Task 2: Deploy user_import as a CronJob via semester-provisioning Helm Chart

**Objective:** Wire the built image into `helmfile/apps/semester-provisioning/` so it deploys as a CronJob that syncs users from IAM API → UCS on a schedule.

**Files:**
- Modify: `helmfile/apps/semester-provisioning/values.yaml` — set correct image, add secrets config
- Modify: `helmfile/apps/semester-provisioning/templates/cronjobs.yaml` — verify template works with user-import entrypoint
- Create: `helmfile/environments/default/semester-provisioning.yaml.gotmpl` (if missing)
- Modify: `helmfile.yaml.gotmpl` — add semester-provisioning release if missing

**Step 1: Review existing cronjobs.yaml template**

Read the template and ensure it supports:
- `provision` mode — sync users from IAM API to UCS
- `deprovision_disable` mode — daily check for users to disable
- `deprovision_delete` mode — daily check for users past grace period

**Step 2: Configure values.yaml with correct image**

Update `values.yaml`:
- `semesterProvisioning.image.repository` → `registry.hrz.uni-marburg.de/opendesk-edu/user-import`
- `semesterProvisioning.image.tag` → `latest`
- Add `extraEnv` for UDM and Keycloak credentials (from secrets)

**Step 3: Deploy via helmfile**

```bash
helmfile -n opendesk-edu apply --skip-deps
# Or direct helm if .gotmpl can't render
helm upgrade --install semester-provisioning helmfile/apps/semester-provisioning \
  --namespace opendesk-edu \
  --set semesterProvisioning.image.repository=registry.hrz.uni-marburg.de/opendesk-edu/user-import
```

**Step 4: Verify pods start successfully**

```bash
kubectl -n opendesk-edu get pods -l app.kubernetes.io/instance=semester-provisioning
kubectl -n opendesk-edu logs -l app.kubernetes.io/instance=semester-provisioning --tail=20
```

**Step 5: Commit**

```bash
git add helmfile/apps/semester-provisioning/
git commit -m "feat: deploy user-import as semester-provisioning CronJob"
```

## Risks & Open Questions

- **Image registry:** Ensure `registry.hrz.uni-marburg.de/opendesk-edu/user-import` exists or create it
- **UDM REST API access:** The CronJob pod needs network access to the UDM API (port 443) — verify network policies in opendesk-edu namespace
- **DFN-AAI registration:** The DFN-AAI test federation registration is a manual process requiring interaction with `www.aai.dfn.de` — may block behind bureaucratic process
- **Helmfile .gotmpl workaround:** The semester-provisioning chart may need direct `helm` upgrade until the .gotmpl rendering issue is resolved (HRZ-specific CoreDNS quirk)
