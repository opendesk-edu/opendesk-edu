# Migrate from MinIO to SeaweedFS

This guide assumes that your `kubectl` is configured with access to a
Kubernetes cluster, the variable `$NAMESPACE` contains the
namespace in which openDesk has been deployed to, and the variable
`$ENVIRONMENT` contains the Helmfile environment to deploy.

It is also assumed that the default hostnames and domain (`domain.tld`) are in
use. If you are using custom hostnames please adjust the `global.hosts` values
and ensure that MinIO and SeaweedFS are not using the same hostname if they are
deployed at the same time.

The migration from MinIO to SeaweedFS **requires a downtime**:

1. Deploy openDesk with MinIO and SeaweedFS enabled:

   ```yaml
   # File: helmfile/environments/$ENVIRONMENT/values.yaml.gotmpl
   debug:
     # Enable debug to deploy MinIO admin console.
     # We will need it the next step
     enabled: true
   apps:
     minio:
       enabled: true
     seaweedfs:
       enabled: true # Default value
   ```

   ```bash
   # Deploy external services (including SeaweedFS)
   pushd helmfile/apps/services-external
   helmfile apply -n "$NAMESPACE" -e "$ENVIRONMENT"
   popd
   ```

   This will add an SeaweedFS deployment that is reachable under
   `objectstorage.domain.tld`. To make migration easier, the application
   credentials are the same as for MinIO.

2. Fetch admin credentials for MinIO and SeaweedFS (to make migration easier,
   SeaweedFS uses the same credentials as MinIO):

      ```bash
      # Get minio admin secret
      kubectl -n "$NAMESPACE" get secret minio \
        -o jsonpath='{.data.root-password}' | base64 -d
      ```

3. Disable S3 application users to prevent further writes:

   This can be done on the admin UI we enabled in the previous step. The MinIO
   admin UI is reachable under <https://objectstore-ui.domain.tld>.

4. Sync data from MinIO to SeaweedFS:

   1. Create rclone config secret:

      First we need the admin credentials for MinIO and SeaweedFS:

      Now we can create a secret containing a rclone config file. Replace the
      `$*_ADMIN_SECRET` endpoint values in the following file with your
      environment specific values:

      ```cfg
      # File: rclone.conf
      [minio]
      type = s3
      provider = Minio
      access_key_id = admin
      secret_access_key = $MINIO_ADMIN_SECRET
      endpoint = https://objectstore.domain.tld

      [seaweedfs]
      type = s3
      provider = SeaweedFS
      access_key_id = admin
      secret_access_key = $MINIO_ADMIN_SECRET
      endpoint = https://objectstorage.domain.tld
      ```

      Create the secret:

      ```bash
      kubectl -n "$NAMESPACE" create secret generic rclone-config \
        --from-file=rclone.conf
      ```

   2. Start migration job:

      ```yaml
      # File: rclone-s3-migration.yaml
      apiVersion: batch/v1
      kind: Job
      metadata:
        name: rclone-s3-migration
      spec:
        backoffLimit: 3
        template:
          metadata: {}
          spec:
            restartPolicy: Never
            containers:
              - image: docker.io/rclone/rclone:sha-c0f03f8
                name: rclone-s3-migration
                # rclone sync flags:
                #   - verbose: increase verbosity
                #   - update: skip files that are newer on the receiver
                #   - checksum: skip based on checksum, not mod-time & size
                #   - inplace: download directly to destination file instead of atomic download to temp/rename
                args:
                  - sync
                  - --verbose
                  - --update
                  - --checksum
                  - --inplace
                  - minio:/
                  - seaweedfs:/
                resources:
                  limits:
                    cpu: 0.1
                    memory: 100Mi
                  requests:
                    cpu: 0.1
                    memory: 100Mi
                volumeMounts:
                  - name: config
                    readOnly: true
                    mountPath: /home/rclone/.rclone.conf
                    subPath: rclone.conf
                securityContext:
                  allowPrivilegeEscalation: false
                  capabilities:
                    drop:
                      - "ALL"
                  privileged: false
                  seccompProfile:
                    type: "RuntimeDefault"
                  readOnlyRootFilesystem: true
                  runAsNonRoot: true
                  runAsUser: 1009
                  runAsGroup: 1009
                  seLinuxOptions: {}
            volumes:
              - name: config
                secret:
                  secretName: rclone-config
      ```

      ```bash
      kubectl -n "$NAMESPACE" create -f rclone-s3-migration.yaml
      ```

      After the migration has finished you can visit
      <https://objectstorage-ui.domain.tld> to check if all the data
      has been migrated (username: `admin`, password: stored in secret, check
      previous step).

5. Delete the MinIO deployment and reconfigure applications to use SeaweedFS
   endpoint and credentials (also disabled debug as it is not needed anymore):

   ```yaml
   # File: helmfile/environments/$ENVIRONMENT/values.yaml.gotmpl
   apps:
     minio:
       enabled: false # Default value, can be ommited
     seaweedfs:
       enabled: true # Default value, can be ommited
   ```

   ```bash
   # Apply changes to the whole deployment
   helmfile apply -e dev

   # Restart the ums-umc-server deployment to reload the S3 configuration
   kubectl rollout restart deployment/ums-umc-server

   # Restart the opendesk-nextcloud-aio deployment to reload the S3 configuration
   kubectl rollout restart deployment/opendesk-nextcloud-aio
   ```
