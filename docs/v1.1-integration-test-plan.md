# openDesk Edu v1.1 Integration Test Plan

## Overview

This test plan validates the complete v1.1 feature set working together in an integrated environment. Tests cover authentication flows, user lifecycle, backchannel logout, and DFN-AAI federation.

## Test Environment

### Requirements

- Kubernetes 1.28+ cluster
- Keycloak 23+ with SAML support
- UCS UDM REST API access
- DFN-AAI test federation access (for Sprint 6.3)
- Test domain: `test.opendesk.example.com`

### Deployed Components

| Component | Version | Purpose |
|-----------|---------|---------|
| Keycloak | 23.x | SSO, SAML/OIDC IdP |
| Moodle | 4.x | LMS with SAML backchannel |
| ILIAS | 9.x | LMS with SAML backchannel |
| BigBlueButton | 2.7.x | Video conferencing |
| OpenCloud | 1.x | File storage (OIDC) |
| Nextcloud | 28.x | File storage (OIDC) |
| Intercom Service | 1.1.x | Email routing |
| User Provisioning | 1.1.x | Lifecycle management |

## Test Suite 1: Authentication Flows

### 1.1 Local Authentication

**Test:** User logs in with local Keycloak credentials.

```bash
# Test command
curl -X POST "https://id.test.opendesk.example.com/realms/opendesk/protocol/openid-connect/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=password&client_id=opendesk-portal&username=testuser&password=Test123!"
```

**Expected:**
- HTTP 200 response
- Valid JWT token with claims
- Token contains: sub, email, name, roles

**Pass Criteria:** Token accepted by all services.

### 1.2 DFN-AAI SAML Authentication

**Test:** User authenticates via DFN-AAI federation.

**Steps:**
1. Navigate to `https://portal.test.opendesk.example.com`
2. Click "Login with Institution"
3. Select test institution from DFN-AAI
4. Authenticate with test credentials
5. Verify redirect back to portal

**Expected:**
- SAML assertion received by Keycloak
- User attributes mapped correctly (ePPN, email, affiliation)
- OIDC token issued with eduGAIN claims
- User can access all services

**Pass Criteria:** Full SSO flow completes in < 5 seconds.

### 1.3 Service-Specific Authentication

**Test:** Direct service access triggers SSO.

**Steps:**
1. Access `https://moodle.test.opendesk.example.com` directly
2. Should redirect to Keycloak
3. Authenticate
4. Redirect back to Moodle with active session

**Expected:** Seamless SSO without re-authentication.

## Test Suite 2: User Lifecycle

### 2.1 User Provisioning

**Test:** Import users from CSV.

```bash
cd scripts/user_import
docker run --rm \
  -e IMPORT_DOMAIN=test.opendesk.example.com \
  -e UDM_API_PASSWORD=secret \
  -e KEYCLOAK_API_PASSWORD=secret \
  -v ./test-users.csv:/data/users.csv \
  opendesk-user-import:latest \
  python provision.py --import_filename /data/users.csv
```

**Expected:**
- Users created in UCS
- SAML identities linked in Keycloak
- Email accounts provisioned
- Admin accounts created (if enabled)

**Pass Criteria:** 100% of users from CSV provisioned successfully.

### 2.2 User Deprovisioning (Phase 1: Disable)

**Test:** Disable users no longer in IAM.

```bash
docker run --rm \
  -e IMPORT_DOMAIN=test.opendesk.example.com \
  -e UDM_API_PASSWORD=secret \
  -e IAM_API_URL=https://iam.test.opendesk.example.com/deprovision_list \
  opendesk-user-import:latest \
  python deprovision_disable.py --dry_run
```

**Expected:**
- Users not in IAM identified
- SAML identities removed
- Groups removed (except system groups)
- Accounts disabled but not deleted

**Pass Criteria:** Disabled users cannot authenticate.

### 2.3 User Deprovisioning (Phase 2: Delete)

**Test:** Delete users after grace period.

```bash
docker run --rm \
  -e IMPORT_DOMAIN=test.opendesk.example.com \
  -e UDM_API_PASSWORD=secret \
  opendesk-user-import:latest \
  python deprovision_delete.py --grace_period_months 0 --dry_run
```

**Expected:**
- Users past grace period identified
- All user data archived
- User accounts permanently deleted
- Admin accounts deleted

**Pass Criteria:** Deleted users fully removed from all systems.

## Test Suite 3: Backchannel Logout

### 3.1 SAML Backchannel Logout (ILIAS, Moodle, BBB)

**Test:** Portal logout terminates SAML service sessions.

**Steps:**
1. Login to portal
2. Open ILIAS, Moodle, BBB in separate tabs
3. Verify authenticated in all services
4. Click logout in portal
5. Wait 30 seconds
6. Check each service tab

**Expected:**
- All services show logged-out state
- Session cookies cleared
- Re-access requires re-authentication

**Pass Criteria:** All sessions terminated within 60 seconds.

### 3.2 OIDC Backchannel Logout (OpenCloud, Nextcloud)

**Test:** Portal logout terminates OIDC service sessions.

**Steps:**
1. Login to portal
2. Open OpenCloud, Nextcloud in separate tabs
3. Verify authenticated
4. Click logout in portal
5. Wait 30 seconds
6. Check each service tab

**Expected:**
- Keycloak sends backchannel POST to `/backchannel_logout`
- Services terminate sessions
- Refresh shows login screen

**Pass Criteria:** All OIDC sessions terminated within 60 seconds.

### 3.3 Partial Logout Failure

**Test:** One service unavailable during logout.

**Steps:**
1. Stop one service (e.g., ILIAS)
2. Login to remaining services
3. Logout from portal
4. Restart stopped service
5. Check session state

**Expected:**
- Available services logout successfully
- Unavailable service session times out naturally
- No errors block user logout

**Pass Criteria:** User sees logout confirmation; unavailable service session expires.

## Test Suite 4: DFN-AAI Federation

### 4.1 Test Federation Authentication

**Test:** Authenticate via DFN-AAI test federation.

**Steps:**
1. Submit SP metadata to https://test.aai.dfn.de/
2. Wait for approval
3. Configure Keycloak with test IdP
4. Test authentication

**Expected:**

- Authentication succeeds

**Pass Criteria:** Successful authentication with test IdP.

**Test:** Verify all eduGAIN attributes mapped correctly.

**Expected OIDC Claims:**

```json
  "sub": "test-user-id",
  "email": "user@testuni.de",
  "name": "Test User",
  "given_name": "Test",
  "family_name": "User",
  "eppn": "user@testuni.de",
  "affiliation": "member@testuni.de",
  "homeOrg": "https://testuni.de"
}
```

**Pass Criteria:** All 5 required + recommended attributes present.

### 4.3 Role-Based Access Control

**Test:** eduPersonAffiliation maps to openDesk roles.

| Affiliation | Expected Role |
|-------------|---------------|
| `student@...` | Student |
| `faculty@...` | Teacher |
| `staff@...` | Admin |
| `member@...` | User |

**Pass Criteria:** Correct role assignment based on affiliation.

## Test Suite 6: Edge Cases

### 6.1 Concurrent Sessions

**Test:** User logged in from multiple devices.

**Steps:**
1. Login from desktop
2. Login from mobile
3. Logout from desktop
4. Check mobile session

**Expected:**
- Desktop logout doesn't affect mobile (by design)
- Both sessions independent

**Pass Criteria:** Sessions managed independently.

### 6.2 Session Timeout

**Test:** Idle session expires.

**Steps:**
1. Login to portal
2. Wait for session timeout (default: 8 hours)
3. Try to access service

**Expected:**
- Session expired
- Re-authentication required

**Pass Criteria:** Expired session rejected.

### 6.3 Certificate Expiry

**Test:** SAML certificate expires.

**Steps:**
1. Set SAML signing cert to expire in past
2. Attempt SAML authentication

**Expected:**
- Authentication fails gracefully
- Clear error message
- No security bypass

**Pass Criteria:** Expired cert rejected.

## Test Suite 7: Load Testing

### 7.1 Concurrent Users

**Test:** 500 concurrent users.

```bash
# Using k6 load testing tool
k6 run scripts/load-test/concurrent-users.js \
  -e USERS=500 \
  -e DURATION=5m \
  -e RAMP_UP=1m
```

**Expected:**
- All users can authenticate
- Average response time < 2 seconds
- Error rate < 1%

**Pass Criteria:** System handles 500 concurrent users.

### 7.2 Login Burst

**Test:** 100 logins in 10 seconds.

**Expected:**
- No authentication failures
- Keycloak handles burst
- No service degradation

**Pass Criteria:** All logins succeed within 30 seconds.
