## Release Process

### Step 1: Create Release Branch

```bash
git checkout main
git pull
git checkout -b release/v1.1
git push origin release/v1.1
```

- [ ] Release branch created
- [ ] Branch protection enabled

### Step 2: Final Verification

```bash
# Run final smoke tests
./scripts/release/smoke-test.sh v1.1.0

# Verify all tests pass
./scripts/release/verify-release.sh v1.1.0
```

- [ ] Smoke tests pass
- [ ] All CI/CD pipelines green

### Step 3: Create Git Tag

```bash
git tag -a v1.1.0 -m "openDesk Edu v1.1.0 Release

Features:
- SAML backchannel logout (ILIAS, Moodle, BBB)
- OIDC backchannel logout (OpenCloud, Nextcloud)
- User provisioning Docker image + runbook
- DFN-AAI federation integration
- Intercom service with ICS routing
- Branded articles with teaser SVGs

Security:
- SOPS secrets management
- CMP sidecar integration
- Backchannel logout for session security

Documentation:
- Complete operations runbooks
- DFN-AAI integration guide
- Integration test plan"

git push origin v1.1.0
```

- [ ] Annotated tag created
- [ ] Tag pushed to all remotes (github, codeberg, opencode-de)

### Step 4: Create GitHub Release

1. Navigate to: https://github.com/opendesk-edu/opendesk-edu/releases/new
2. Tag: `v1.1.0`
3. Title: `openDesk Edu v1.1.0`
4. Copy changelog from `CHANGELOG.md`
5. Attach release artifacts
6. Mark as "Latest release"
7. Publish

- [ ] GitHub release created
- [ ] Release notes complete
- [ ] Artifacts attached

### Step 5: Deploy to Production

```bash
# Update ArgoCD application
kubectl apply -f kubernetes/release-v1.1.yaml

# Monitor deployment
kubectl rollout status deployment/opendesk-portal -n opendesk
```

- [ ] Production deployment initiated
- [ ] Health checks pass
- [ ] Monitoring shows normal operation

### Step 6: Post-Deployment Verification

```bash
# Run production smoke tests
./scripts/release/prod-smoke-test.sh

# Verify key functionality
curl -I https://portal.opendesk.example.com
curl -I https://id.opendesk.example.com
```

- [ ] Production smoke tests pass
- [ ] All services accessible
- [ ] No errors in logs

```bash
git push origin :refs/tags/v1.1.0
```
